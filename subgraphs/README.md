# Romain — federation subgraphs (cft-federation-server layout)

This tree mirrors the per-subgraph package style from [`cft-federation-server/subgraphs/*`](../../../Loop/cft-federation-server/subgraphs): each domain has its own folder with `package.json`, `Dockerfile`, `src/<domain>.graphql`, and `index.ts`.

The big difference: **all five subgraphs share the same Payload + Hono runtime image.** Each container/process selects a single pruned Federation 2 view via `PAYLOAD_LAMBDA_SUBGRAPH=<domain>` (see `_shared/subgraph/payloadSubgraphByDomain.ts:buildFederatedSubgraphForDomain`), so we don't run five separate Payload bootstraps and five DB connection pools for one CMS. The shared runtime (the Hono app, the Payload+Federation glue, the Lambda handler) lives in `_shared/`; there is no top-level `src/`.

| Domain    | Default port (local) | Docker hostname (compose) | Owns                                                                                  |
| --------- | -------------------- | ------------------------- | ------------------------------------------------------------------------------------- |
| `users`   | 4001                 | `users:4000`              | `User`                                                                                |
| `groups`  | 4002                 | `groups:4000`             | `Group`, `GroupMember`, `GroupPost`, `GroupComment`, `GroupPostLike`, `GroupCommentLike` |
| `search`  | 4003                 | `search:4000`             | `@payloadcms/plugin-search` index + queries                                           |
| `content` | 4004                 | `content:4000`            | `Page`, `Post`, `Media`, `Category`, plus form/redirect/seo/nested-docs surfaces      |
| `system`  | 4005                 | `system:4000`             | `payload-folders`, `payload-locked-documents`, `payload-preferences`, `payload-migrations`, `payload-jobs`, `Header`, `Footer` |

## Files in each subgraph dir

- `package.json` — informational; `start` runs `tsx index.ts`.
- `Dockerfile` — builds against the **monorepo root** (so `romainRetreatCMS` and `romainRetreatServer` both end up in the image), sets `PAYLOAD_LAMBDA_SUBGRAPH`, and exposes the per-domain port.
- `src/<domain>.graphql` — generated by `yarn export:subgraph-sdl` (mirror of `supergraph/payload-sdl/<domain>.graphql`). Used as the SDL source by `supergraph/schema/local.yaml` and friends.
- `index.ts` — calls `runSubgraph('<domain>')` from `_shared/bootstrap.ts` to start the shared Hono server.

## Shared

- `_shared/bootstrap.ts` — sets `PAYLOAD_LAMBDA_SUBGRAPH` + a default `PORT`, then awaits `startGraphqlServer()` from `_shared/standaloneServer.ts`.
- `_shared/{app,server,lambda,standaloneServer,load-env}.ts`, `_shared/{subgraph,federation,graphql}/*` — the shared Hono + Payload + Federation 2 runtime that previously lived in `romainRetreatServer/src/`.
- `domains.ts` — canonical `PAYLOAD_SUBGRAPH_DOMAINS` list, type guard, and `PAYLOAD_SUBGRAPH_DEV_PORTS`.

## Local development (cft-style)

```sh
# From romainRetreatServer
make up-subgraphs       # docker compose up users groups search content system  (ports 4001..4005)
make compose            # rover supergraph compose --config supergraph/schema/local.yaml > supergraph/supergraph.local.graphql
make run-router         # apollo router --supergraph supergraph/supergraph.local.graphql
make smoke              # quick sanity query

# Or skip Docker:
yarn dev:subgraph:users   # only :4001
yarn dev                  # unified app (3002 by default) — handy when iterating Payload
```

## AWS / GraphOS

Same code is shipped to **per-domain** Lambda Function URLs via SAM (`yarn deploy:lambda all`) with `SubgraphMode=<domain>` (= `PAYLOAD_LAMBDA_SUBGRAPH`). Routing URLs are listed in `supergraph/schema/aws.introspect.yaml` (or `aws.unified-file.yaml` when introspection is disabled).
